# Shifa - Therapy & Mental Wellness Platform for Tunisia
## Comprehensive Strategic Project Plan

---

## 1. Context & Vision

**Problem:** Tunisia faces a significant mental health treatment gap. With ~1 psychiatrist per 60,000 people, stigma around mental health, and limited access in rural areas, millions lack affordable, culturally appropriate mental health support.

**Solution:** "Shifa" (شفاء — meaning "healing" in Arabic) — a bilingual therapy and mental wellness platform built as a PWA + mobile app, designed specifically for Tunisian cultural context, language, and market dynamics.

**Target Users:**
- Tunisians aged 18–45 (primary), 45+ (secondary)
- University students, young professionals, women, diaspora Tunisians
- Licensed therapists, counselors, and peer support volunteers

## 2. Technical Architecture

### 2.1 Tech Stack (Replit-Compatible)

| Layer | Technology | Rationale |
|-------|-----------|-----------|
| **Frontend** | Next.js 14+ (React) | SSR/SSG for PWA performance, excellent i18n support, Replit-native |
| **UI Library** | Tailwind CSS + Radix UI | RTL-first styling, accessible components |
| **Mobile Wrapper** | Capacitor.js | Wraps PWA into native iOS/Android with access to native APIs (camera, push notifications, biometrics) |
| **Backend** | Next.js API Routes + Node.js | Unified codebase, runs on Replit |
| **Database** | Supabase (PostgreSQL) | Free tier, real-time subscriptions, Row Level Security, auth built-in |
| **Auth** | Supabase Auth + OAuth | Email/password, Google, Facebook, phone OTP (Twilio) |
| **Real-time Messaging** | Supabase Realtime + WebSockets | Encrypted chat, typing indicators, read receipts |
| **Video/Audio Calls** | Daily.co or Jitsi Meet API | HIPAA-aware, low-bandwidth optimization for Tunisian networks |
| **File Storage** | Supabase Storage (S3-compatible) | Encrypted at rest, for documents/recordings |
| **Push Notifications** | Firebase Cloud Messaging (FCM) | Cross-platform push |
| **Payments** | Flouci API + Konnect (D17) + Stripe (international) | Tunisia-specific payment gateways |
| **Email** | Resend or Brevo (Sendinblue) | Transactional + marketing, French/Arabic templates |
| **Analytics** | PostHog (self-hosted option) or Mixpanel | Privacy-first, GDPR-compliant |
| **CI/CD** | Replit Deployments + GitHub Actions | Auto-deploy from Replit |
| **Monitoring** | Sentry (error tracking) + UptimeRobot | Production reliability |

### 2.3 PWA Configuration

- **Service Worker:** Workbox for offline caching (journal entries, self-help content)
- **Manifest:** Installable on home screen with Shifa branding

## 4. Internationalization (i18n) & Localization

### 4.1 Language Support

| Language | Code | Direction | Usage Context |
|----------|------|-----------|---------------|
| Modern Standard Arabic | `ar` | RTL | Formal content, professional communications |
| French | `fr` | LTR | Alternative UI, professional/academic users |
| Tunisian Darija | `ar-TN` | RTL | Peer support, chatbot, informal content |

### 4.2 Implementation

- **Library:** `next-intl` for Next.js i18n
- **URL Strategy:** `/ar/dashboard`, `/fr/dashboard` (path-based)
- **Translation Files:** JSON per locale, organized by feature namespace
- **RTL Layout:** Tailwind `dir="rtl"` with `rtl:` variant classes
- **Font Stack:**
  - Arabic: Noto Sans Arabic, Amiri (for formal), Tajawal (modern)
  - French: Inter, system-ui
- **Number/Date Formatting:** Intl API with `ar-TN` and `fr-TN` locales
- **Darija Considerations:**
  - No standardized written form — use phonetic Arabic script
  - Build custom Darija glossary for mental health terms
  - User-contributed translations with moderation
- **Seamless Switching:** Language toggle in header, persisted in user preferences, no page reload

### 4.3 Cultural Adaptation

- Imagery reflecting Tunisian culture (Mediterranean, diverse skin tones, hijab-optional)
- Culturally sensitive mental health terminology
- Islamic-informed therapeutic approaches available as option
- Gender-sensitive UX (option to filter for same-gender therapist)
- Ramadan-aware scheduling and content

## 5. UX/UI Design Considerations

### 5.1 Design Principles
1. **Privacy-first:** Anonymous mode default, minimal data display
2. **Stigma-reduction:** Warm, non-clinical aesthetic; avoid medical imagery
3. **Accessibility:** WCAG 2.1 AA compliant
4. **Low-bandwidth friendly:** Skeleton screens, lazy loading, compressed assets
5. **Thumb-friendly:** Bottom navigation, large tap targets (48px minimum)

### 5.2 Color Palette
- Primary: Calming teal/turquoise (#0D9488) — culturally neutral, soothing
- Secondary: Warm amber (#F59E0B) — welcoming, optimistic
- Background: Soft cream (#FFFBF0) for light mode, deep navy (#0F172A) for dark
- Success: Green (#10B981)
- Crisis: Warm red (#EF4444) for SOS, not alarming

### 5.3 Key Screens
1. **Home Dashboard:** Mood check-in prompt, upcoming session, daily tip, quick actions
2. **Explore:** Therapist cards, self-help resources, support groups
3. **Chat:** WhatsApp-like interface (familiar to Tunisian users)
4. **Profile:** Settings, mood history, journal, payment history
5. **SOS:** Full-screen crisis support with large, clear options

### 5.4 RTL-Specific Design
- Mirrored layouts (navigation, icons, swipe gestures)
- Bidirectional text handling for mixed Arabic/French content
- Arabic-first placeholder text
- Right-aligned form fields in RTL mode
- Mirrored progress indicators and carousels

## 8. Compliance & Regulatory

### 8.1 Tunisian Data Protection
- **Organic Law No. 2004-63** (Protection of Personal Data): Tunisia's primary data privacy law
- **INPDP** (Instance Nationale de Protection des Données Personnelles): Must register data processing activities
- Requirements:
  - Explicit user consent for data collection
  - Right to access, rectify, and delete personal data
  - Data storage location transparency
  - Breach notification obligations

### 8.2 Health Data Specifics
- Sensitive health data requires **explicit written consent**
- Encrypted storage and transmission mandatory
- Access logs for all health data queries
- Data minimization: collect only what's necessary
- Retention policy: configurable, default 5 years with option to delete

### 8.3 Therapist Licensing
- Verify credentials against Tunisia's Ministry of Health registry
- Psychologists: Licence en Psychologie from accredited Tunisian university
- Psychiatrists: Medical degree + specialization
- Counselors: Define certification pathway (currently unregulated — opportunity)
- Display verification status prominently

### 8.4 Implementation
- End-to-end encryption for all clinical communications (AES-256)
- At-rest encryption for database (Supabase built-in)
- HTTPS everywhere (TLS 1.3)
- Session recordings: explicit consent + encrypted storage
- Anonymized analytics (no PII in analytics pipeline)
- GDPR-adjacent practices (for future EU expansion and diaspora users)
- Regular security audits and penetration testing
- Privacy policy and terms of service in Arabic and French

## 10. Challenges & Mitigation

| Challenge | Impact | Mitigation |
|-----------|--------|------------|
| **Mental health stigma** | Low adoption | Anonymity features, Darija content to normalize, influencer partnerships |
| **Low purchasing power** | Revenue pressure | Freemium model, student discounts, pay-what-you-can, B2B subsidies |
| **Internet connectivity** | Poor UX in rural areas | PWA offline mode, audio-only sessions, low-bandwidth optimization |
| **Limited therapist supply** | Bottleneck for growth | Peer support model, train counselors, attract diaspora therapists |
| **Payment infrastructure** | Friction in payments | Multiple local gateways (Flouci, D17, Sobflous), telco billing exploration |
| **Arabic NLP immaturity** | Poor Darija understanding | Build custom Darija datasets, partner with Tunisian NLP researchers |
| **Regulatory ambiguity** | Legal risk | Proactive INPDP registration, legal counsel, over-comply on data protection |
| **Therapist trust/quality** | Reputation risk | Strict verification, user reviews, session quality monitoring |
| **Competition** | Market share | First-mover in Tunisia-specific solution, Darija = key differentiator |
| **Replit limitations** | Scaling constraints | Start on Replit, plan migration to Railway/Vercel at scale |
